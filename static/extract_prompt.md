**任务目标**
请分析提供的图片，提取其中的目录信息。针对每个目录项，需提取标题（t）、页码（n）以及所属层级（level）。

**输出格式要求**
1. 仅输出JSON数据，严格禁止包含任何其他文本或说明。
2. JSON结构需按层级组织，具体层级数量视图片内容而定。请参考以下两级目录的示例结构：
```json
{
  "level_1": [
    {"t": "一级标题1", "n": 1},
    {"t": "一级标题2", "n": 25}
  ],
  "level_2": [
    {"t": "二级标题1", "n": 5},
    {"t": "二级标题2", "n": 8}
  ]
}
```

**内容提取规则**
1. **完整性与筛选**：不要遗漏任何目录项目。带有页码的条目必然属于目录条目，无论其具体内容为何。需忽略与正文无关的条目，如`前言`或`第x版前言`。
2. **无页码条目的判定**：部分条目可能没有直接显示页码，应基于具体语义判断其是否属于目录的一部分，不可单纯依赖格式规范度。若出现大批量无页码的目录（如用于描述某章节下的关键词集合），应当予以忽略。
3. **所见即所得**：严格按照规范提取页面上真实存在的信息，严禁进行任何添加。若页面以4.5.1开头，绝对禁止自行推测或补充第四章及4.5节的标题，应当直接提取当前可见的全部内容。
4. **忠于原文**：提取标题时严禁擅自增添、删减或修改。必须严格保留原始目录的语言，若存在繁体中文、英文等非简体中文文字，应当直接提取原始内容，禁止将其翻译为简体中文。
5. **符号替换**：禁止使用带圈的数字（如①），应当替换为常规数字（如1）。
6. **标题页码分割**：页码可能紧跟在标题后，此时应注意不要将页码算作标题的一部分。

**层级判定规则**
1. **视觉特征优先**：应严格根据图片呈现的视觉特征（如颜色、字体、文字大小等）判定目录层级，不可依赖语义进行判断。
2. **规避层级判定错误**：
   * **节与子节的层级关系**：例如2.4（节）与2.4.1（子节）绝对不可处于同一目录层级，子节的层级必须比节低一级。
   错误示例（节与子节同级）：
   ```json
   {
     "text": "2.4 典型全控型器件",
     "number": 25,
     "level": 2
   },
   {
     "text": "2.4.1 门极可关断晶闸管",
     "number": 26,
     "level": 2
   }
   ```

**页码处理规则**
1. **缺失页码估算**：当部分目录条目缺失页码时，请根据其附近的内容进行合理推算。例如，第1篇的页码丢失，但第1篇第1章的页码为2，则推测第1篇的页码为2。
2. **篇章页码与层级处理示例**：
   错误示例（未估算页码，且将篇与章混入同级）：
   ```json
   "level_1": [
       {"t": "xx篇", "n": null}, 
       {"t": "xx章", "n": 1},
       {"t": "xx章", "n": 5} 
     ]
   ```
   正确示例（合理推算页码，且层级正确分离）：
   ```json
   "level_1": [
       {"t": "xx篇", "n": 1} 
     ],
   "level_2": [
       {"t": "xx章", "n": 1},
       {"t": "xx章", "n": 5}
     ]
   ```

**空格与排版规则**
1. **纯中文目录**：确保目录项的标题和页码之间仅保留1个空格，除此之外禁止加入任何空格。
   * 正确：`第1章 自动控制概述`；`第2章 超前滞后校正与PID校正`
   * 错误：`第 1章 自动控制概述`；`第1 章自动控制概述`；`第2章 超前滞后校正与 PID校正`；`第2章 超前滞后校正与PID 校正`
2. **纯英文目录**：遵循标准英语语法，单词与数字之间保留正常空格。
3. **混合目录**：中文部分适用中文处理规则，英文部分适用英文语法规则。
