# 问题排查方案

## 程序运行部分

遇到任何问题，请先点击右上角的`通义千问服务状态测试`，若服务状态异常，可参见[错误信息](https://help.aliyun.com/zh/model-studio/error-code?spm=a2c4g.11186623.help-menu-2400256.d_2_13_0.57704c44CQVCqj)进行处理；若服务状态正常但无法得到正确的PDF文件，请咨询`uijm2004@outlook.com`以获取进一步支持。

如果第一步执行时间非常长且反复失败，一般是由于`目录起始页`和`目录结束页`的差值特别大导致的。请仔细查看`3.2 上传 PDF 并处理`的图片示例，以确保你对这两个数字的理解正确。

## 环境配置部分

如果自动环境配置脚本`windows_install.bat`或`macos_install.sh`运行时出现问题，请手动配置运行环境：

1. Python 版本 == 3.11（其他版本未测试）
2. poppler 版本 == 24（Windows用户可直接前往`setup_documents`文件夹下获取）
3. requirements.txt 中的全部依赖

详细教程如下。环境配置完毕后，请回到项目主页，从 API-KEY 配置部分（2.2.1 的第 2 条或 2.2.2 的第 4 条）继续。

### Windows

1. 从 [Python 官方网站](https://www.python.org/downloads/release/python-3119/)根据自己的系统版本选择并下载安装 Python，选择时注意即使是 ARM 架构的 Windows 电脑也要选择 x64 的 Python；安装时注意勾选"Add Python to PATH"选项
2. 安装完成后，Win + R 输入 cmd 打开命令提示符，输入以下内容以验证安装。若输出版本号表明安装成功。
    ```cmd
    python --version
    pip --version
    ```
3. 从 [Poppler 官方网站](https://poppler.freedesktop.org)下载并安装 Poppler。
4. Win + R 输入 cmd 打开命令提示符，然后打开项目目录
    ```cmd
    cd path/to/autoContents-main
    ```
5. 创建并激活虚拟环境
    ```cmd
    python -m venv .venv
    venv\Scripts\activate
    ```
6. 安装依赖
    ```cmd
    pip install -r requirements.txt
    ```

### macOS

1. 打开“终端”应用
2. 输入以下命令以安装 Homebrew，安装过程中选择中国的镜像源（一般选 1 或 2）
    ```zsh
    /bin/zsh -c "$(curl -fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)"
    ```
3. 安装完成后验证安装，若正常输出版本号表明安装成功。
    ```zsh
    brew --version
    ```
4. 另开一个“终端”窗口，安装 Python 3.11
    ```zsh
    brew install python@3.11
    ```
5. 确认 Python 安装成功
    ```zsh
    source ~/.zshrc
    python --version
    ```
6. 安装 Poppler
    ```zsh
    brew install poppler
    ```
7. 确认 Poppler 安装成功
    ```zsh
    source ~/.zshrc
    poppler --version
    ```
8. 创建并激活虚拟环境
    ```zsh
    python -m venv .venv
    source .venv/bin/activate
    ```
9. 安装依赖
    ```zsh
    pip install -r requirements.txt
    ```
